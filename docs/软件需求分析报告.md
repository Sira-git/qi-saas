## 2 产品/项目概述

### 2.1 产品/项目背景

在 AI 时代，企业面临着海量的政策信息、行业动态、竞争对手情报和中标数据。传统的手工整理方式效率低下，信息滞后，难以支撑企业快速决策。

"企情察"平台应运而生，通过 AI 技术自动整合五大维度数据（政策、行业、友商、中标、情报），一键生成日报、周报、月报，帮助企业实时掌握市场动态，抢占先机。

### 2.2 产品/项目建设目标

1. **智能报告生成**：一键生成日报、周报、月报，自动整合五大维度数据，告别繁琐的手工整理
2. **政策追踪分析**：实时跟踪适配企业的政策内容，解读政策要点及对企业的影响
3. **竞品动态监测**：持续追踪竞品动向，分析竞争格局，量化差距
4. **中标数据洞察**：全面掌握市场中标信息，发现商机，提升中标率

### 2.3 产品/项目亮点

- **AI 驱动**：利用 AI 技术自动分析和整合数据，减少人工干预
- **多维度报告**：支持日报、周报、月报多种报告类型，满足不同场景需求
- **政策智能订阅**：支持自定义订阅策略，精准推送匹配的政策信息
- **企业空间管理**：支持多企业空间、子账号管理、权益分配
- **实时预警**：重要政策、友商动态实时预警，不错过任何关键信息

### 2.4 竞品分析

| 竞品名称 | 优势 | 劣势 | 我们的差异化 |
|---------|------|------|-------------|
| 传统情报服务 | 数据权威 | 价格高、更新慢 | 实时、性价比高 |
| 手工整理 | 灵活定制 | 效率低、易出错 | 自动化、准确率高 |
| 通用 BI 工具 | 功能全面 | 学习成本高 | 专注企业情报、上手简单 |

### 2.5 假定和约束

#### 2.5.1 假定

1. 用户具备基本的互联网使用能力
2. 用户企业已注册并拥有合法的经营资质
3. 数据来源合法合规，已获取必要的授权
4. 网络环境稳定，能够支持实时数据更新

#### 2.5.2 约束

1. **技术约束**：
   - 前端采用 Vue 3 + Vite + Ant Design Vue
   - 后端 API 接口需遵循 RESTful 规范
   - 支持主流浏览器（Chrome、Safari、Edge）

2. **业务约束**：
   - 免费版限制 1 个子账号、3 个竞品监控
   - 付费版按套餐等级分配权益
   - 数据更新频率受限于数据源接口

3. **时间约束**：
   - 报告生成时间：日报<5 分钟，周报<15 分钟，月报<30 分钟
   - 政策推送延迟：<10 分钟

### 3.3 用户端功能需求

#### 3.3.1 注册/登录模块

**功能描述**：
- 用户通过手机号 + 验证码注册账号
- 注册时填写企业名称和所属行业
- 登录时输入手机号和验证码

**界面要求**：
- 左右分栏布局，左侧展示平台特点，右侧表单
- 背景色：整体#f8faff，左侧#f0f5ff
- 输入框：高度 40px，圆角 8px，边框 1px solid #d9d9d9
- 按钮：高度 44px，蓝色#1677ff

**交互逻辑**：
1. 注册时发送验证码，60 秒倒计时
2. 密码至少 8 位
3. 注册成功后自动创建企业空间（默认免费版）
4. 登录默认账号：13800138000 / 123456

**数据字段**：
| 字段名 | 类型 | 必填 | 说明 |
|-------|------|------|------|
| phone | string | 是 | 手机号，11 位 |
| code | string | 是 | 验证码，6 位数字 |
| password | string | 是 | 密码，至少 8 位 |
| companyName | string | 是 | 企业全称 |
| industry | string | 是 | 所属行业 |

#### 3.3.2 报告管理模块

##### 3.3.2.1 报告列表

**功能描述**：
- 展示所有报告，支持筛选和搜索
- 显示报告标签（政策、行业、友商、中标数量）
- 支持报告类型 Tab 切换（全部、日报、周报、月报）

**界面要求**：
- 卡片式列表展示
- 报告类型图标背景色：日报蓝色、周报绿色、月报紫色
- 异常状态卡片：淡黄色背景#fffef0
- 标签：橙色（#fff7e6 背景，#d46b08 文字）

**筛选条件**：
| 筛选项 | 类型 | 选项 |
|-------|------|------|
| 关键词 | 输入框 | 报告名称 |
| 报告类型 | 下拉框 | 日报、周报、月报 |
| 报告状态 | 下拉框 | 正常、异常 |
| 日期范围 | 日期选择器 | 开始日期 - 结束日期 |

**交互逻辑**：
1. Tab 切换时更新报告列表
2. "全部报告"数量显示总数，不随筛选变化
3. 点击卡片进入报告详情
4. 支持收藏、下载报告

##### 3.3.2.2 报告设置

**功能描述**：
- 配置自动报告生成规则（日报、周报、月报）
- 手动选择日期生成报告
- 展示企业基本信息，支持跳转配置

**自动报告**：
- 默认展示三个卡片（日报、周报、月报），初始禁用
- 点击"配置"弹出设置窗口
- 日报：每隔 N 天生成
- 周报：选择每周几生成
- 月报：选择每月几号生成
- 显示下次生成时间

**手动报告**：
- 报告类型：卡片式选择（日报/周报/月报）
- 日期选择：
  - 日报：选择具体日期
  - 周报：选择日期，提示自动生成该周报告
  - 月报：选择月份（Picker="month"）
- 日期限制：只能选择当天及之前

**企业信息联动**：
- 顶部展示企业基本信息（简介、区域、业务范围、友商）
- 简介展示两行，虚线分隔
- 友商展示总数及前 2 个
- 支持跳转配置

### 4.3 安全性需求

#### 认证与授权

1. 用户登录必须验证手机号和验证码
2. 接口请求必须携带 Token 验证
3. 支持 IP 白名单限制（管理端）
4. 子账号权限隔离，只能访问授权资源

#### 数据安全

1. 密码加密存储（bcrypt）
2. 敏感信息脱敏展示（手机号中间 4 位）
3. 接口参数校验，防止 SQL 注入
4. 文件上传限制类型和大小

#### 访问安全

1. URL 不存在时返回通用 404 页面，不泄露敏感信息
2. 禁止 DELETE、PUT、OPTIONS、TRACE、HEAD 等方法
3. 目录采用非常规名称，避免使用 admin、config 等
4. 跨域请求需要验证 Origin

#### 日志审计

1. 记录用户关键操作（登录、报告生成、配置修改）
2. 管理端操作日志完整记录
3. 日志保留至少 6 个月
