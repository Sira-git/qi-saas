## 三、用户端（C 端）功能规划

### 3.1 账号体系（重构后）

**架构模型**：主账号（自然人）→ 企业空间 → 子账号

#### 3.1.1 核心概念

| 概念 | 说明 |
|------|------|
| **主账号** | 自然人账号，通过手机号/邮箱注册，是系统的根节点 |
| **企业空间** | 企业级数据隔离单元，绑定套餐，拥有独立的情报数据、报告、子账号 |
| **子账号** | 挂靠在企业空间下的操作账号，权限由空间管理员分配 |
| **空间管理员** | 子账号角色之一，可管理该空间内的子账号、监控规则、报告等 |

#### 3.1.2 主账号注册/登录

- ⏳ 手机号/邮箱注册主账号
- ⏳ 主账号登录（账号密码）
- ⏳ 首次登录后引导创建企业空间
- ⏳ 选择套餐（基础版/高级版）
- ⏳ 创建成功，进入该企业空间

#### 3.1.3 子账号激活

- ⏳ 主账号在企业空间内创建子账号（填写子账号邮箱）
- ⏳ 系统发送邀请链接到子账号邮箱
- ⏳ 子账号点击邀请链接，设置独立密码
- ⏳ 激活成功，自动绑定到该企业空间

#### 3.1.4 角色权限（MVP 3 级）

| 角色 | 说明 | 权限范围 |
|------|------|----------|
| **主管理员** | 主账号本人 | 管理所有企业空间、子账号、套餐 |
| **空间管理员** | 从子账号名额中分配 | 管理本空间内的子账号、监控规则、报告 |
| **普通子账号** | 基础操作账号 | 查看/生成报告、收藏夹、企业档案 |

#### 3.1.5 企业空间规则

- 一个主账号可创建多个企业空间
- 仅主账号可在不同空间间切换；子账号绑定单一空间不可切换
- 每个空间独立绑定套餐，拥有独立的情报数据和子账号

#### 3.1.6 空间切换

- 仅主账号可见顶部空间切换器
- 切换空间后，Dashboard 数据实时切换为对应空间数据
- 子账号登录直接进入绑定空间，无切换器
