# 企情察 - 账号体系重构需求变更

> 项目编号：#7
> 变更日期：2026-05-06
> 变更类型：架构重构（账号体系、角色权限、计费模式）
> 状态：需求确认，待实施

---

## 一、变更概述

### 1.1 变更原因

原设计采用「一企一户」模式（一个用户直接绑定一个企业账号），无法满足以下业务场景：
- 集团企业需要管理多个子企业的独立情报空间
- 同一企业内部分享账号，需要不同权限的子账号
- 企业需要独立套餐订阅和计费

### 1.2 变更影响

| 影响范围 | 影响等级 | 说明 |
|----------|----------|------|
| 账号体系 | 🔴 架构重构 | 从「用户-企业」改为「主账号-空间-子账号」三层架构 |
| 角色权限 | 🔴 权限重构 | 从 2 级（用户/管理员）改为 3 级（MVP）/4 级（完整版） |
| 数据隔离 | 🟡 数据库调整 | 所有数据表增加 `space_id` 字段 |
| 计费模式 | 🟡 计费系统复杂化 | 从按企业账号改为按空间套餐 + 阶梯折扣 |
| 前端页面 | 🔴 部分重构 | 注册流程、空间切换器、子账号管理页面 |

## 二、核心架构设计

### 2.1 关系模型

```
主账号（自然人/老板）
├── 企业空间 A（绑定套餐 = 收费单元 + 数据隔离单元）
│   ├── 空间管理员（从子账号名额中分配）
│   ├── 普通子账号 × N
│   └── 情报数据、监控规则、报告...
├── 企业空间 B（绑定套餐）
│   ├── 空间管理员
│   └── 普通子账号 × N
└── 集团观察员（旗舰版专属，跨空间只读汇总）→ 暂缓，第二期实现
```

### 2.2 核心概念

| 概念 | 说明 |
|------|------|
| **主账号** | 自然人账号，通过手机号/邮箱注册，是系统的根节点 |
| **企业空间** | 企业级数据隔离单元，绑定套餐，拥有独立的情报数据、报告、子账号 |
| **子账号** | 挂靠在企业空间下的操作账号，权限由空间管理员分配 |
| **空间管理员** | 子账号角色之一，可管理该空间内的子账号、监控规则、报告等 |
| **集团观察员** | 子账号角色之一（旗舰版），跨空间只读汇总（暂缓） |

## 三、角色权限体系

### 3.1 MVP 第一期：3 级角色

| 角色 | 说明 | 权限范围 |
|------|------|----------|
| **主管理员** | 主账号本人 | 管理所有企业空间、子账号、套餐 |
| **空间管理员** | 从子账号名额中分配 | 管理本空间内的子账号、监控规则、报告 |
| **普通子账号** | 基础操作账号 | 查看/生成报告、收藏夹、企业档案 |

### 3.2 完整版：4 级角色

| 角色 | 说明 | 权限范围 | 套餐要求 |
|------|------|----------|----------|
| **主管理员** | 主账号本人 | 全量权限 | 所有套餐 |
| **空间管理员** | 子账号角色 | 本空间管理权限 | 所有套餐 |
| **普通子账号** | 基础操作 | 查看/生成/收藏 | 所有套餐 |
| **集团观察员** | 跨空间只读汇总 | 多空间数据汇总查看 | 旗舰版 |

## 四、MVP 范围确认

### 4.1 决策清单

| 维度 | 决策 |
|------|------|
| 子账号登录 | 主账号发送邀请链接 → 子账号点击激活 → 设置独立密码登录 |
| 角色层级 | 3 级：主管理员 / 空间管理员 / 普通子账号（集团观察员暂缓） |
| 多空间视图 | 无聚合看板，主账号切换空间看对应 Dashboard |
| 套餐差异 | 仅 3 项：子账号数量、监控竞品数量、预警功能 |
| 收费模式 | 展示页 + 选择提交（无实际支付、无折扣计算） |
| 工商信息 | 创建空间时不强制验证 |

### 4.2 第一期 MVP 功能清单

| 功能 | 说明 |
|------|------|
| ✅ 主账号注册/登录 | 手机号/邮箱注册，作为根节点 |
| ✅ 企业空间创建 | 简化版（暂不验证工商信息，仅填名称 + 行业） |
| ✅ 空间切换器 | 仅主账号可见，顶部下拉切换 |
| ✅ 子账号管理 | 主账号可创建/删除子账号，绑定到空间 |
| ⚠️ 角色权限 | 先实现 3 级（去掉集团观察员） |
| ⚠️ 套餐体系 | 先实现 2 级（基础版/高级版） |

### 4.3 第二期规划

| 功能 | 说明 |
|------|------|
| 集团观察员 | 旗舰版专属，多空间汇总看板 |
| 阶梯折扣 | 计费系统自动计算 |
| 子账号超额增购 | 独立计费 |
| 工商信息验证 | 空间创建时强制验证 |
| API 接口/私有化 | 旗舰版专属 |

## 五、流程设计

### 5.1 注册/登录流程（重构）

**原流程**：
```
手机号注册 → 直接成为企业用户
```

**新流程**：
```
手机号/邮箱注册 → 主账号
    ↓
首次登录后引导创建企业空间
    ↓
选择套餐（基础版/高级版）
    ↓
创建成功，进入该企业空间
```

### 5.2 子账号激活流程

```
主账号在企业空间内创建子账号
    ↓
主账号填写子账号邮箱
    ↓
系统发送邀请链接到子账号邮箱
    ↓
子账号点击邀请链接
    ↓
设置独立密码
    ↓
激活成功，自动绑定到该企业空间
```

### 5.3 首页/仪表盘逻辑（调整）

**原逻辑**：Dashboard 显示单个企业数据

**新逻辑**：
```
主账号登录 → 看到空间切换器 + 当前空间数据
    ↓
切换空间 → Dashboard 切换为对应空间数据

子账号登录 → 直接进入绑定空间，无切换器
    ↓
显示当前空间 Dashboard
```

|------|-----------|-------------|---------|-----------|-----------|
| 基础版 | 3 个 | 5 个 | ❌ | ❌ | ❌ |
| 高级版 | 10 个 | 20 个 | ✅ | ❌ | ❌ |

|------|------|
| `main_accounts` | 主账号（手机号、邮箱、密码） |

## 八、前端页面调整

### 8.1 需要重构的页面

| 页面 | 调整内容 |
|------|----------|
| 注册页 | 改为注册主账号（手机号/邮箱） |
| 登录页 | 支持主账号和子账号登录 |
| 首页 | 添加空间切换器（仅主账号可见） |
| Dashboard | 数据按当前空间隔离 |
| 企业档案 | 改为空间信息配置 |

### 8.2 新增页面

| 页面 | 路由 | 功能 |
|------|------|------|
| 空间创建 | `/space/create` | 创建企业空间 + 选择套餐 |
| 子账号管理 | `/space/sub-accounts` | 创建/删除/邀请子账号 |
| 套餐选择 | `/space/packages` | 展示套餐 + 选择提交 |

### 8.3 管理端调整

**现有**：管理企业账号（增删改查）

**新设计**：
```
管理端需要管理：
- 主账号列表（查看所有主账号）
- 企业空间列表（查看所有空间 + 套餐状态）
- 子账号列表（按空间过滤）
- 账单管理（按空间生成账单）
```

## 九、与现有设计差异对比

| 维度 | 现有设计 | 新设计 | 影响等级 |
|------|----------|--------|----------|
| 账号体系 | 一企一户，手机号直接注册企业 | 主账号（自然人）→ 创建企业空间 → 子账号挂靠空间 | 🔴 |
| 角色层级 | 用户/管理员 两级 | 主管理员/空间管理员/普通子账号/集团观察员 四级 | 🔴 |
| 数据隔离 | 企业级别 | 企业空间级别（物理隔离） | 🟡 |
| 收费单元 | 按企业账号 | 按企业空间（独立套餐 + 阶梯折扣） | 🟡 |
| 切换逻辑 | 用户可切换绑定企业 | 仅主账号可切换空间；子账号绑定单一空间不可切换 | 🔴 |
